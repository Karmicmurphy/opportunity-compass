# Research Workflow

This document locks the workflow so the user is not left guessing where to go or what to connect.

## Tool roles

### ChatGPT

Command center, web researcher, translator, card builder, spec writer, and GitHub safety guard.

ChatGPT should do public web research first whenever possible.

### GitHub

Public repo for safe project files only:

- specs
- templates
- fake/sample data
- public-safe summaries
- builder handoff docs
- prototype code later

### Google Drive

Private file bridge for raw material:

- screenshots
- copied post text
- research dumps
- spreadsheets
- private notes that should not go public

### Obsidian

Private project brain and long-term notes.

### Canva

Mockups and visual layout drafts.

### Hugging Face

Later research for models, datasets, classifiers, and open-source ideas.

### Lovable

Optional prototype shortcut if credits are available.

## Important correction

Fiverr and Reddit are research sources, not required ChatGPT connectors in this setup.

The user should not have to guess whether to connect them.

ChatGPT should search public sources first. The user only needs to provide screenshots or copied text when a source is locked inside an app, account, login wall, or unavailable to the web tool.

## Research target

Build the first evidence pile around:

- beginner-friendly AI gigs
- AI freelancing services
- data annotation/task work
- remote work scam risks
- small business AI help
- prompt work
- AI content cleanup
- automation planning
- overlooked local business tasks

## Processing rule

Raw research should be converted into Opportunity Compass cards.

Each card needs:

- title
- source
- category
- beginner fit
- verdict
- estimated pay range
- estimated time
- tools needed
- skills needed
- hidden buyer pain
- what this really means
- why beginners miss it
- smallest paid offer
- suggested first message
- questions to ask
- deliverable checklist
- red flags
- green flags
- salvage pick candidate

## Public safety rule

Do not commit raw screenshots, private emails, private account data, private names, addresses, payment data, API keys, or copied paid-platform content to GitHub.

Commit only summaries, fake examples, public-safe notes, and original Opportunity Compass cards.
